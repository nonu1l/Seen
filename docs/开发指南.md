# Seen Java AI / Agent 开发者指南

这份文档面向有 Java Spring 基础、想了解 Seen AI 助手实现的读者。这里只保留启动配置、阅读路线和关键链路；细节直接看源码。

## 可以学到什么

- Spring AI `ChatClient` 与 Tool Calling。
- 自主 Agent 如何按用户输入选择搜索、推荐、标记、取消标记和读取长期记忆工具。
- 作品卡片、记录、快照和 `requestId` 如何串起一次 AI 操作。
- 网页搜索、页面抓取、LLM 提取、Bangumi 匹配组成的搜索管道。
- LLM 输出清理、工具失败可见化、停止任务和单例运行锁。
- Token 用量、缓存命中和 AI 调用观测。

## 构建环境

`Java 21` / `Node.js 22.17.1` / `Maven 3.6.3`

如果遇到编译失败或项目无法启动，可以先对照以上版本检查本地环境。

## 快速配置

### Cloudflare Worker 反向代理

```bash
cd cf-worker/bangumi-proxy
npm install
npx wrangler login
npx wrangler deploy
```

项目里的 Worker 只代理 Bangumi API 和图片；AI 网络搜索由后端按设置页选择的 Serper 或 Tavily 调用，也可以选择“不使用”关闭外部搜索。

### AI 与搜索 Key

- AI Provider API Key：支持 OpenAI 兼容接口，例如 DeepSeek、MiniMax 等。
- Serper API Key：[serper.dev](https://serper.dev/)
- Tavily API Key：[app.tavily.com](https://app.tavily.com/)

启动后进入 `/settings`，在设置页填写 AI Base URL、模型、API Key、Bangumi 代理地址和搜索源；这些运行时配置会保存到数据库。

### 本地启动

后端：

```bash
mvn spring-boot:run
```

前端：

```bash
cd frontend
npm install
npm run dev
```

默认地址：

- 后端：`http://localhost:8081`
- 前端：`http://localhost:5173`
- Token 用量追踪：`http://localhost:8081/admin/token-usage`

> 发布前执行 `npm run build`，前端静态资源会输出到 `src/main/resources/static`。

## Agent 架构

```
用户输入
    │
    ▼
ConversationService
  ├─ 保存用户消息与助手占位消息
  ├─ 创建 requestId、单例运行锁和 SSE 状态监听
  └─ 调用 AutonomousAgentService
          │
          ▼
Autonomous Agent
  ├─ searchBangumi / searchLocal / getWorkState
  ├─ findWorks → SearchPipeline
  ├─ presentWorks → 生成候选卡片
  ├─ markWork → 保存记录并生成可撤销卡片
  ├─ unmarkWork → 安全检查后取消本地记录
  ├─ readUserMemory
  └─ searchWeb(可关闭) / fetchWeb
          │
          ▼
最终回复 + 本轮 requestId 下的卡片
```

建议按这个顺序看：

| 顺序 | 文件 | 重点 |
|---|---|---|
| 1 | `controller/ConversationController.java` | AI 对话、停止、卡片保存和撤回入口 |
| 2 | `service/ConversationService.java` | 会话、SSE、单例锁、停止、requestId 串联 |
| 3 | `agent/AutonomousAgentService.java` | 自主 Agent 调用模型和注册工具 |
| 4 | `agent/tool/AiToolRegistry.java` | 暴露给模型的工具列表和动态搜索工具开关 |
| 5 | `agent/tool/AiAutonomousTools.java` | findWorks、presentWorks、markWork、unmarkWork 等工具门面 |
| 6 | `service/AiWorkOperationService.java` | AI 卡片、record 写入、快照和撤回恢复 |
| 7 | `agent/SearchPipeline.java` | 推荐 / 搜索 / 描述找片的多步搜索管道 |
| 8 | `resources/prompts/*.st` | 自主 Agent、标题提取、匹配校验等 prompt |
| 9 | `config/TokenUsageAdvisor.java` | Token 用量、缓存命中和调用节点记录 |

### SearchPipeline 搜索管道

```
用户输入
  │
  ├─ 1. generateKeywords → 3组搜索关键词
  ├─ 2. searchWeb → 按设置选择 Serper / Tavily；关闭或无结果时进入直访兜底
  ├─ 3. fetchWeb → 抓取搜索结果页面，或由 LLM 选择公开榜单 URL 直接抓取
  ├─ 4. extractTitles → 从页面内容提炼片名
  ├─ 5. searchBangumi → 并发匹配 Bangumi 条目
  ├─ 6. 去重：片名和 subjectId 去重
  ├─ 7. validateMatches → 校验候选与用户需求是否匹配
  └─ 8. 聚合最多5条候选；全空时生成失败说明
```

## 开发建议

- 先跑普通影视记录功能，确认 Bangumi 搜索、标记和详情页可用。
- 再打开 AI，先测试普通问答和本地记录查询。
- 然后测试“标记某部作品”和“取消某部作品标记”，观察卡片、record 和快照。
- 最后测试推荐或描述找片，重点看 `SearchPipeline`、搜索源配置和 Token 用量页。
- 初次修改建议从 prompt、小工具函数或设置项开始，不要直接改整条 Agent 链路。
