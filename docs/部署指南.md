# Seen 部署指南

1. 部署 Cloudflare Worker 反向代理
2. 申请 AI 需要的 API Key
3. 拉取 Docker 镜像并启动

如果不需要 AI 助手，只完成第 1 步和第 3 步。

## 第一步：部署 Cloudflare Worker 反向代理


> `xxx.workers.dev` 域名在部分网络环境下可能会受到 DNS 解析污染，导致即使 Worker 部署成功但无法正常访问。更稳的做法是把你自己的域名托管到 Cloudflare，再把自定义域名绑定到这个 Worker。

如果觉得部署 Worker 太麻烦，也可以先使用提供的公开 [反向代理地址](反向代理地址.md)，暂时跳过第一步。
### 1. 注册 Cloudflare

1. 打开 [Cloudflare 官网](https://www.cloudflare.com/)。
2. 点击注册账号。
3. 使用邮箱完成注册和验证。
4. 登录后进入 Cloudflare 控制台。

### 2. 安装 Node.js

在本机安装 Node.js 20 或更高版本。可以从 [Node.js 官网](https://nodejs.org/) 下载 LTS 版本。

安装后在终端执行：

```bash
node -v
npm -v
```

能看到版本号就说明安装好了。

### 3. 登录 Cloudflare Wrangler

下载当前项目到本地解压，并在项目目录执行：

```bash
cd cf-worker/bangumi-proxy
npm install
npx wrangler login
```

执行 `npx wrangler login` 后，浏览器会打开 Cloudflare 授权页面。登录并允许授权。

### 4. 部署 Worker

继续执行：

```bash
npx wrangler deploy
```

部署成功后，终端会显示一个类似这样的地址：

```text
https://bangumi-proxy.xxxxxx.workers.dev
```

把这个地址保存下来，后面在设置页填写。注意地址末尾不要加 `/`。


## 第二步：申请 AI 需要的 API Key

如果你不需要 AI 助手，可以跳过这一节，启动后在设置页关闭 AI 助手。

如果你需要 AI 助手，需要准备：

- **DeepSeek API Key**（必选）：负责大模型对话、意图识别、推荐文案。
- **Serper 或 Tavily API Key**（可选）：负责 AI 搜索网页内容。使用搜索能力时需要在设置页选择其中一个搜索源并填写对应 API Key。

### 申请 DeepSeek API Key（必选）

1. 打开 [DeepSeek API 平台](https://platform.deepseek.com/)。
2. 注册或登录账号。
3. 进入 API Keys 页面。
4. 创建新的 API Key。
5. 复制生成的 key。

启动后进入设置页，在“AI 助手”中填写 API Key、Base URL 和模型名称并保存。

### 2. 申请搜索 API Key

如果需要 AI 网络搜索，可以选择 Serper 或 Tavily 任意一个搜索源。

1. 打开 [Serper.dev](https://serper.dev/)。
2. 注册或登录账号。
3. 进入 Dashboard。
4. 找到 API Key。
5. 复制 API Key。

也可以打开 [Tavily](https://app.tavily.com/) 注册并复制 Tavily API Key。

启动后进入设置页，在“搜索与数据源”中选择 Serper 或 Tavily，并填写对应 API Key。

## 第三步：拉取 Docker 镜像并启动


只想当影视 / 番剧记录系统使用，可以这样启动：

```bash
docker pull youmiepie/seen:latest

docker run -d \
  --name seen \
  -p 8081:8081 \
  -v /opt/seen-data:/app/data \
  -e SPRING_SQL_INIT_MODE=always \
  youmiepie/seen:latest
```

> 数据库文件将保存在宿主机的 `/opt/seen-data/media.db`，方便查看和备份。
> 首次启动前请先执行 `mkdir -p /opt/seen-data`。

启动后打开：

```text
http://localhost:8081
```

### 使用 AI 助手

容器启动后打开设置页：

```text
http://localhost:8081/settings
```

在“AI 助手”中填写 DeepSeek API Key、Base URL、模型名称并保存；在“搜索与数据源”中填写 Bangumi 代理地址，必要时选择 Serper 或 Tavily 并填写对应 API Key。

> 设置保存到数据库后立即生效，不需要重启 Docker 容器。

启动后打开：

```text
http://localhost:8081
```

## 常用命令

查看日志：

```bash
docker logs -f seen
```

停止容器：

```bash
docker stop seen
```

重新启动：

```bash
docker start seen
```

删除容器：

```bash
docker rm -f seen
```

保留数据的关键是这个挂载：

```bash
-v /opt/seen-data:/app/data
```

不要随便删除 `/opt/seen-data` 目录，否则 SQLite 数据库也会被删除。

## 环境变量说明

| 变量 | 必填 | 说明 |
|---|---|---|
| `SPRING_SQL_INIT_MODE` | **必填** | 必须设为 `always`，否则数据库基础数据不会初始化 |

AI、搜索源、Serper API Key、Bangumi 代理地址等运行时设置都保存在数据库中，请在设置页保存。

## 常见问题

### 搜索剧片搜不出任何内容

通常是数据库基础数据未初始化导致。请确认启动命令中包含了：

```bash
-e SPRING_SQL_INIT_MODE=always
```

部署后验证：

```bash
docker exec -it seen sqlite3 /app/data/media.db "SELECT * FROM subject_type;"
```

应该输出 `2|动画` 和 `6|真人`，如果为空则说明初始化未生效。

### 页面能打开，但是没有封面图

通常是 Bangumi 代理地址没有配置正确。请确认：

- Worker 已经部署成功。
- 设置页中的 Bangumi 代理地址是完整地址，例如 `https://seen.slimrip.com` 或你自己部署的 Worker 地址。
- 地址末尾没有 `/`。

### AI 助手不能用

请确认：

- 设置页中已开启 AI 助手，并且 API Key 已保存
- `SPRING_SQL_INIT_MODE=always`（数据库未初始化也会导致 AI 搜索工具无法工作）
- 容器日志里没有 API key 或网络错误（`docker logs -f seen`）

### 不想用 AI，可以吗

可以。在设置页关闭 AI 助手后，Seen 仍然可以作为普通影视 / 番剧记录系统使用。
