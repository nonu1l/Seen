# Seen 部署指南

1. 部署 Cloudflare Worker 反向代理
2. 申请 AI 需要的 API Key
3. 拉取 Docker 镜像并启动

如果不需要 AI 助手，只完成第 1 步和第 3 步。

## 第一步：部署 Cloudflare Worker 反向代理


> `xxx.workers.dev` 域名在部分网络环境下可能会受到 DNS 解析污染，导致即使 Worker 部署成功但无法正常访问。更稳的做法是把你自己的域名托管到 Cloudflare，再把自定义域名绑定到这个 Worker。

如果觉得部署 Worker 太麻烦，也可以先使用提供的公开 [反向代理地址](反向代理地址.md)，暂时跳过第一步。
### 1. 注册 Cloudflare

1. 打开 [Cloudflare 官网](https://www.cloudflare.com/)。
2. 点击注册账号。
3. 使用邮箱完成注册和验证。
4. 登录后进入 Cloudflare 控制台。

### 2. 安装 Node.js

在本机安装 Node.js 20 或更高版本。可以从 [Node.js 官网](https://nodejs.org/) 下载 LTS 版本。

安装后在终端执行：

```bash
node -v
npm -v
```

能看到版本号就说明安装好了。

### 3. 登录 Cloudflare Wrangler

下载当前项目到本地解压，并在项目目录执行：

```bash
cd cf-worker/bangumi-proxy
npm install
npx wrangler login
```

执行 `npx wrangler login` 后，浏览器会打开 Cloudflare 授权页面。登录并允许授权。

### 4. 部署 Worker

继续执行：

```bash
npx wrangler deploy
```

部署成功后，终端会显示一个类似这样的地址：

```text
https://bangumi-proxy.xxxxxx.workers.dev
```

把这个地址保存下来，后面 Docker 启动时会用到。注意地址末尾不要加 `/`。


## 第二步：申请 AI 需要的 API Key

如果你不需要 AI 助手，可以跳过这一节，并在启动 Docker 时设置：

```bash
-e AI_ENABLED=false
```

如果你需要 AI 助手，需要准备两个 key：

- DeepSeek API Key：负责大模型对话、意图识别、推荐文案。
- Serper API Key：负责 AI 搜索网页内容。

### 1. 申请 DeepSeek API Key

1. 打开 [DeepSeek API 平台](https://platform.deepseek.com/)。
2. 注册或登录账号。
3. 进入 API Keys 页面。
4. 创建新的 API Key。
5. 复制生成的 key。

Docker 启动时配置为：

```bash
-e LLM_API_KEY=REPLACE_WITH_DEEPSEEK_API_KEY
-e LLM_BASE_URL=https://api.deepseek.com
-e LLM_MODEL=deepseek-v4-flash
```

### 2. 申请 Serper API Key

1. 打开 [Serper.dev](https://serper.dev/)。
2. 注册或登录账号。
3. 进入 Dashboard。
4. 找到 API Key。
5. 复制 API Key。

Docker 启动时配置为：

```bash
-e SEARCH_PROVIDER=serper
-e SERPER_API_KEY=REPLACE_WITH_SERPER_API_KEY
```

如果没有 Serper key，项目会尝试使用 DuckDuckGo 代理搜索，但 AI 搜索结果可能不如 Serper 稳定。

## 第三步：拉取 Docker 镜像并启动

### 不使用 AI 助手

只想当影视 / 番剧记录系统使用，可以这样启动：

```bash
docker pull youmiepie/seen:latest

docker run -d \
  --name seen \
  -p 8081:8081 \
  -v seen-data:/app/data \
  -e BANGUMI_PROXY=https://seen.slimrip.com \
  -e AI_ENABLED=false \
  youmiepie/seen:latest
```

启动后打开：

```text
http://localhost:8081
```

### 使用 AI 助手

需要 AI 助手时这样启动：

```bash
docker pull youmiepie/seen:latest

docker run -d \
  --name seen \
  -p 8081:8081 \
  -v seen-data:/app/data \
  -e BANGUMI_PROXY=https://seen.slimrip.com \
  -e AI_ENABLED=true \
  -e LLM_API_KEY=REPLACE_WITH_DEEPSEEK_API_KEY \
  -e LLM_BASE_URL=https://api.deepseek.com \
  -e LLM_MODEL=deepseek-v4-flash \
  -e SEARCH_PROVIDER=serper \
  -e SERPER_API_KEY=REPLACE_WITH_SERPER_API_KEY \
  youmiepie/seen:latest
```

启动后打开：

```text
http://localhost:8081
```

## 常用命令

查看日志：

```bash
docker logs -f seen
```

停止容器：

```bash
docker stop seen
```

重新启动：

```bash
docker start seen
```

删除容器：

```bash
docker rm -f seen
```

保留数据的关键是这个卷：

```bash
seen-data:/app/data
```

不要随便删除 Docker volume，否则 SQLite 数据库也会被删除。

## 环境变量说明

| 变量 | 必填 | 说明 |
|---|---|---|
| `BANGUMI_PROXY` | 是 | Cloudflare Worker 地址，用于访问 Bangumi API、图片和 DuckDuckGo Lite |
| `AI_ENABLED` | 否 | 是否开启 AI 助手，`true` 或 `false` |
| `LLM_API_KEY` | 使用 AI 时必填 | DeepSeek API Key |
| `LLM_BASE_URL` | 使用 AI 时建议填写 | DeepSeek API 地址，默认 `https://api.deepseek.com` |
| `LLM_MODEL` | 使用 AI 时建议填写 | 大模型名称 |
| `SEARCH_PROVIDER` | 使用 AI 时建议填写 | 推荐 `serper` |
| `SERPER_API_KEY` | 使用 AI 搜索时必填 | Serper API Key |

## 常见问题

### 页面能打开，但是没有封面图

通常是 `BANGUMI_PROXY` 没有配置正确。请确认：

- Worker 已经部署成功。
- `BANGUMI_PROXY` 是完整地址，例如 `https://seen.slimrip.com` 或你自己部署的 Worker 地址。
- 地址末尾没有 `/`。

### AI 助手不能用

请确认：

- `AI_ENABLED=true`
- `LLM_API_KEY` 已配置
- `SERPER_API_KEY` 已配置
- 容器日志里没有 API key 或网络错误

### 不想用 AI，可以吗

可以。设置：

```bash
-e AI_ENABLED=false
```

Seen 仍然可以作为普通影视 / 番剧记录系统使用。
