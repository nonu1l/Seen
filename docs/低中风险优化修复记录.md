  - `usesThinkingToggle`

  - `chatCompletionsUrl`

结果：

- Serper / DuckDuckGo / Bangumi 测试接口路径与响应结构不变。

### Jackson 注解复核

涉及文件：

- `src/main/java/com/nonu1l/media/model/dto/*.java`

处理结果：

- 已复核 `com.fasterxml.jackson.annotation.JsonInclude` 残留。
- 本地 Maven 依赖中可确认存在 `com.fasterxml.jackson.core:jackson-annotations`，但未发现可直接替换的 `tools.jackson.annotation.JsonInclude` 类。
- 为避免引入不可编译的包名，本轮没有机械迁移或删除这些注解。

后续建议：

- 等用户执行 Java 编译后，如果这些注解确实触发编译或运行问题，再选择：
  - 显式保留 Jackson annotations 依赖；
  - 或移除部分 `JsonInclude`，接受 null 字段输出变化；
  - 或确认正确的 Jackson 3 注解包后统一替换。

### 前端小风险修复

改动文件：

- `frontend/src/api/proxy.ts`
- `frontend/src/components/AppLayout.tsx`
- `frontend/src/pages/AiPage.tsx`
- `frontend/src/api/client.ts`

怎么改：

- `setBangumiProxy("")` 时同步删除 `localStorage.bgm_proxy`，避免清空后读回旧代理。
- `AppLayout` 将外部上下文 API 从 `setOnReset` 改为 `registerReset`，内部使用 `setOnReset(() => fn)` 避免 React 函数式 updater 误用。
- `AiPage` 注册 reset 回调时直接传 `ai.reset`，并清理自动滚动的 timeout。
- API 请求失败时优先读取 JSON 中的 `message` / `error`，非 JSON 响应保留 fallback。

结果：

- AI 页 reset 注册语义更清晰。
- 清空 Bangumi 代理不会被本地缓存覆盖。
- toast 能展示更具体的后端错误。
